# SECURITY.md \u2014 OpenClaw Hardening Checklist (Part 15)\n\n_This workspace is hardened for production use per Part 15 \u2014 Infrastructure Hardening._\n\n## Semantic Approval Categories (Task Brain, 2026.3.31-beta.1+)\n\n**Why:** Name-based allowlists (\"allow bash\") fail because malicious tools can register as `bash_v2` or `bash-improved`. Semantic categories (\"allow execution.*\") are immune to this attack.\n\n**Current policy:**\n\n| Category | Current | Meaning |\n|----------|---------|----------|\n| `read-only.memory` | ASK | Search vault, read memory files |\n| `read-only.web` | ASK | Web search, fetch URLs |\n| `read-only.fs` | ASK | Read files on disk |\n| `execution.shell` | ASK | Run shell commands |\n| `execution.code` | ASK | Execute code (Python, Node, etc.) |\n| `execution.cron` | ASK | Schedule background jobs |\n| `write.fs.workspace` | ASK | Write files inside workspace |\n| `write.fs.outside-workspace` | **DENY** | Write outside workspace (hardened) |\n| `write.git` | ASK | Commit, push, change remotes |\n| `control-plane.skills` | **DENY** | Install, update, remove skills |\n| `control-plane.flows` | **DENY** | Cancel, modify running tasks |\n| `control-plane.auth` | **DENY** | Change auth config |\n\n## Secrets Management\n\n### How to do it RIGHT\n\n```bash\n# Set secrets in environment BEFORE starting gateway\nexport ANTHROPIC_API_KEY=\"sk-ant-...\"\nexport CEREBRAS_API_KEY=\"csk-...\"\nexport GITHUB_TOKEN=\"ghp_...\"\n\n# Then start gateway\nopenclaw start\n```\n\n**In prompts/code:** Reference via interpolation, never hardcode:\n```json\n{ \"apiKey\": \"${ANTHROPIC_API_KEY}\" }\n```\n\n### What NOT to do\n\n\u274c Don't put secrets in `openclaw.json`  \n\u274c Don't write API keys to `AGENTS.md` or `SOUL.md`  \n\u274c Don't commit auth profiles to git  \n\u274c Don't paste credentials into memory files  \n\u274c Don't log secrets to session transcripts  \n\n### Approval Redaction (2026.4.15+)\n\nWhen an approval prompt shows a tool call with secrets, they're automatically redacted:\n```\nDo you approve this tool call?\nTool: create_api_key\nArgs: api_key=\"sk-***\"\n```\n\n## Compaction Model Hardening\n\n**Why:** The default compaction model (Gemini Flash) rate-limits easily, causing crash loops when sessions get Never use Gemini Flash for compaction\n- ??? Use Cerebras (cheap, generous rate limits) or Anthropic Sonnet\n- ??? Run compaction on a separate API key from the main agent\n- ??? Set explicit `reserveTokens` under the model's context window\n\n## ClawHub Skills Policy (Part 23)\n\n**Install rules:**\n\n```json\n{\n  \"skills\": {\n    \"autoUpdate\": false,\n    \"allowList\": [\n      \"openclaw-official/*\",\n      \"trusted-author/skill-name@v1.2.3\"\n    ]\n  }\n}\n```\n\n**Before installing any skill:**\n- [ ] Read the SKILL.md source (not just the README)\n- [ ] Check the author's reputation on ClawHub\n- [ ] Pin to a git tag or commit, never a branch\n- [ ] Verify it's not a typo-squatted copy (openclw-official vs openclaw-oficial)\n\n**Known malicious skill families:**\n- TrojanOpenClaw PolySkill family (1,184+ variants, Feb 2026)\n- Atomic Stealer (39 macOS-targeting skills, Jan 2026)\n- See [Part 23 \u2014 ClawHub](../openclaw-optimization-guide/part23-clawhub-skills-marketplace.md)\n\n## File Permissions & Ownership\n\n```bash\n# Workspace files\nchmod 700 ~/.openclaw/workspace       # Only owner can read\nchmod 600 ~/.openclaw/workspace/*.md  # No world read\nchmod 600 ~/.openclaw/auth-profiles.json\n\n# Config\nchmod 600 ~/.openclaw/openclaw.json\n\n# Memory (contains learning about user's work)\nchmod 700 ~/.openclaw/memory\nchmod 700 ~/.openclaw/workspace/vault\n```\n\n## Gateway Hardening\n\n```json\n{\n  \"gateway\": {\n    \"bind\": \"127.0.0.1\",\n    \"controlUi\": {\n      \"allowInsecureAuth\": false,\n      \"allowedOrigins\": [\n        \"http://localhost:18789\",\n        \"http://127.0.0.1:18789\"\n      ]\n    },\n    \"auth\": {\n      \"mode\": \"token\",\n      \"hotReload\": true\n    }\n  }\n}\n```\n\n**Checklist:**\n- ??? Check Task Brain flows for any calls to it: `openclaw flows list --filter tool=<skill-name>`\n3. Review those flows for exfiltration: did they write files? make API calls?\n4. Audit secrets: rotate any tokens that were in memory during the contamination window\n\n**If you suspect credential leakage:**\n1. Stop the gateway\n2. `grep -r \"sk-\\|gsk-\" ~/.openclaw` — find all mentions\n3. Rotate all exposed tokens immediately (Anthropic, Cerebras, GitHub, etc.)\n4. Review Task Brain audit log: `openclaw flows show <flow-id>` for each suspicious flow\n\n**If approval policy was bypassed:**\n1. Stop the gateway immediately\n2. Check `openclaw --version` — upgrade if you're below 2026.3.31-beta.1\n3. Review all flows since the last known-good time\n4. Consider rolling back to a pre-incident backup\n\n---\n\n**See also:**\n- [Part 15 \u2014 Infrastructure Hardening](../openclaw-optimization-guide/part15-infrastructure-hardening.md)\n- [Part 23 \u2014 ClawHub Skills](../openclaw-optimization-guide/part23-clawhub-skills-marketplace.md)\n- [Part 24 \u2014 Task Brain Control Plane](../openclaw-optimization-guide/part24-task-brain-control-plane.md)\n
